Reading through my Twitter feed, I came across a technical post about Twitter REST APIs - Basically guiding you through the login screen for your account and fetching the updated feed (for that moment).

What I am set to achieve here is that I want to create an Android App (using Kotlin), which resembles Twitter.

If I were to replicate Twitter App's UI, I might require a couple of more weeks than the number of days I assigned for this project. But since the goal of this project was to understand the major API calls and Android outlook of them, my efforts were only to cover the following -

  • A user should be able to login (exiting account) to thier Twitter account.
  • When a user logs in, he/she should be able to see the updated feed of their account for that timestamp.
  • A user should be able to tweet and it should reflect in their feed.
  • Implement "pull-to-refresh" for the feed.

(Android) App Development

Twitter REST API

Twitter provides a developer console from where you can create a new project and utilize their limited service offerings. For instance, they allow you to ping (GET request) 500k times for the duration of 15 days, and it gets reset after that. This limited access ensures the reliable and scalable API service that developers can rely on.

Image credit : Twitter Developer Portal

OAuth 2.0 Standard for Login

I used this as a authorization framework. Once I have logged into my Twitter developer console, it provides me a security token which I can use in my App (a project in Android Studio) to have API requests authorized.

APIKey.properties file from Android Studio

Pull-to-refresh functionality

There are certain view groups I have used in this Android project, and a Recyclerview with SwipeRefreshLayout is one of them. I am attaching the guide I used to achieve this functionality here, but in simpler words, SwipeRefreshLayout is a class Android offers to achieve "pull-to-refresh" UX concept. And RecyclerView is used when you are going to 'recycle' the use of a view (xml), which you already created, for a list of other items.
